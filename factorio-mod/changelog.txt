Version: 0.14.4
Date: 2026-06-29
  Bugfixes:
    - fac_building_place: clear obstacles ONLY when placement is actually blocked (was
      destroying trees/rocks even when placement would fail anyway for other reasons).
    - fac_building_place: pick up lying items by inserting the real LuaItemStack (preserves
      quality/count) and only destroy the ground item if it was fully collected (no item loss).

Version: 0.14.3
Date: 2026-06-29
  Features:
    - fac_building_place now autonomously CLEARS the build site before placing: trees,
      rocks (simple-entity), and lying items (item-entity, collected into inventory).
    - If the companion stands on the build footprint it steps aside (find_non_colliding_position)
      so it no longer blocks its own placement. Only a genuine post-clear failure returns "Cannot place".
---------------------------------------------------------------------------------------------------
Version: 0.14.1
Date: 2026-06-26
  Bugfixes:
    - fac_companion_spawn no longer requires a connected player: spawns near a
      player if one exists, else at the map spawn/origin via
      find_non_colliding_position. Enables companion control on a HEADLESS
      server (no client connected) - required for autonomous RL training.

Version: 0.14.0
Date: 2026-06-26
  Changes:
    - Factorio 2.1 compatibility (factorio_version 2.1, verified on 2.1.8)
  Features:
    - Autonomous build pipeline: fac_building_place_start approaches a safe
      position near the target, clears trees/rocks at the footprint, then
      places - no external pathfinding needed
    - fac_building_place auto-clears trees/rocks before placing
  Bugfixes:
    - Fixed scheduler crash: SOLID_TYPES contained invalid prototype types
      ('chest', 'steam-engine') so find_entities_filtered raised inside the
      unguarded on_nth_tick handler, killing ALL companion queues on any
      build attempt. Use valid types (generator, container, ...)
    - Wrapped every tick subsystem in pcall so one failure can no longer stop
      the whole scheduler
    - Restored stuck-detection + perpendicular obstacle-bypass walking
